# Hallucination Detection and Mitigation in Abstractive Text Summarization

An NLP research project that evaluates and reduces factual hallucinations in summaries generated by **BART, T5, and PEGASUS**.

The project uses a Natural Language Inference model, **BART-Large-MNLI**, to determine whether every generated summary sentence is supported by the source document. Unsupported sentences are corrected using evidence-grounded rewriting and re-evaluated for factual consistency.

## Project Overview

Abstractive summarization models can produce fluent summaries containing facts that are unsupported by or contradictory to the original document.

This project implements a complete pipeline for:

- Generating abstractive summaries
- Detecting hallucinated sentences
- Selecting the most faithful summary candidate
- Finding supporting evidence from the source document
- Rewriting unsupported sentences
- Re-evaluating the corrected summary
- Comparing summary quality before and after mitigation

## Models Used

### Summarization Models

- `facebook/bart-large-cnn`
- `t5-base`
- `google/pegasus-cnn_dailymail`

### Hallucination Detection Model

- `facebook/bart-large-mnli`

## Datasets

The pipeline was evaluated across five domains:

| Dataset | Domain |
|---|---|
| CNN/DailyMail | News |
| XSum | News |
| PubMed | Biomedical research |
| arXiv | Scientific research |
| Multi-LexSum | Legal documents |

## Methodology

### 1. Summary Generation

BART, T5, and PEGASUS generate candidate summaries using beam-search decoding.

### 2. Source Document Chunking

Long source documents are divided into overlapping chunks:

- Chunk size: 400 words
- Overlap: 50 words

### 3. Sentence-Level NLI Evaluation

Each generated summary is divided into sentences.

Every summary sentence is compared with the source-document chunks and classified as:

- Entailment
- Neutral
- Contradiction

Neutral and contradictory sentences are treated as potentially hallucinated.

### 4. Candidate Selection

Multiple candidate summaries are generated for each document.

The summary with the fewest unsupported sentences is selected for mitigation.

### 5. Hallucination Mitigation

For every unsupported sentence:

1. The most relevant evidence chunk is retrieved.
2. The sentence is rewritten using source-supported evidence.
3. The rewritten sentence is checked again with BART-MNLI.
4. Successfully entailed sentences replace the original sentences.
5. Unresolved contradictory sentences may be removed.

## Evaluation Metrics

The generated and mitigated summaries are evaluated using:

- ROUGE-1
- ROUGE-2
- ROUGE-L
- BERTScore F1
- NLI-based hallucination rate
- Sentence-level hallucination score

## Key Results

The mitigation pipeline reduced hallucination while largely preserving summary quality.

Examples from the 100-document experiments:

| Dataset and Model | Before Mitigation | After Mitigation |
|---|---:|---:|
| CNN/DailyMail – BART | 0.29 | 0.14 |
| arXiv – PEGASUS | 0.62 | 0.37 |

BART achieved the strongest overall BERTScore performance across the evaluated datasets, while PEGASUS showed particularly high hallucination on out-of-domain scientific documents.
